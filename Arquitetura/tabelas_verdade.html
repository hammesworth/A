O que é uma Tabela-Verdade?

Uma Tabela-Verdade é uma ferramenta matemática usada para mapear sistematicamente todas as combinações de valores lógicos de entrada para o seu resultado lógico correspondente. Em essência, ela é a "prova definitiva" do comportamento de uma expressão booleana ou de um circuito lógico. Ela não deixa espaço para ambiguidades: para um conjunto específico de entradas, a saída será sempre a mesma.

A beleza da tabela-verdade está em sua simplicidade e poder. Ela nos permite analisar e prever o funcionamento de um circuito antes mesmo de construí-lo, sendo uma ferramenta indispensável no design e na depuração de sistemas digitais.

A Estrutura de uma Tabela-Verdade

Toda tabela-verdade possui uma estrutura bem definida:

  • Colunas de Entrada: Representam todas as variáveis da expressão (A, B, C, etc.).
  • Colunas Intermediárias: Opcionais, mas extremamente úteis para resolver expressões complexas. Elas mostram o resultado de sub-expressões.
  • Coluna de Saída: Mostra o resultado final da expressão booleana para cada combinação de entradas.
  • Linhas: Cada linha representa uma única combinação possível de valores para as variáveis de entrada.

Quantas Linhas Eu Preciso? A Fórmula 2^n

O número de linhas em uma tabela-verdade é determinado pelo número de variáveis de entrada. A regra é simples: se você tem n variáveis, terá 2^n linhas (combinações de entrada possíveis).

  • 1 variável (A): 2^1 = 2 linhas (0, 1)
  • 2 variáveis (A, B): 2^2 = 4 linhas (00, 01, 10, 11)
  • 3 variáveis (A, B, C): 2^3 = 8 linhas (000, 001, ..., 111)
  • 4 variáveis (A, B, C, D): 2^4 = 16 linhas.

Perceba que o número de combinações dobra a cada nova variável adicionada. Para preencher as colunas de entrada sistematicamente, a coluna mais à direita alterna 0, 1, 0, 1... A próxima alterna 0, 0, 1, 1... A seguinte alterna 0, 0, 0, 0, 1, 1, 1, 1... e assim por diante. Isso garante que todas as combinações sejam listadas sem repetições ou omissões.

Construindo Tabelas-Verdade: Passo a Passo

Vamos construir tabelas para expressões, começando pelas mais simples e avançando para as mais complexas.

Exemplo 1: Expressão Simples com NOT e AND -> F = ¬A · B

  1. Identificar Variáveis: Temos 2 variáveis (A e B). Portanto, teremos 2^2 = 4 linhas.
  2. Listar Entradas: Criamos colunas para A e B e listamos todas as 4 combinações.
  3. Colunas Intermediárias: A expressão tem uma parte negada (¬A). É uma boa prática criar uma coluna para ¬A e calcular seus valores invertendo a coluna A.
  4. Calcular a Saída Final: Agora, calculamos a coluna final (F) fazendo a operação AND entre a coluna intermediária ¬A e a coluna de entrada B. Lembre-se, AND (·) só é 1 quando ambos os operandos são 1.
A B ¬A F = ¬A · B

Exemplo 2: Expressão Complexa -> F = (A + B) ⊕ (B · C)

Aqui, a abordagem de colunas intermediárias se torna crucial.

  1. Identificar Variáveis: 3 variáveis (A, B, C). Teremos 2^3 = 8 linhas.
  2. Listar Entradas: Criamos e preenchemos as colunas A, B e C.
  3. Colunas Intermediárias:
    • Primeiro, criamos uma coluna para o primeiro parêntese: (A + B).
    • Depois, criamos outra para o segundo parêntese: (B · C).
  4. Calcular a Saída Final: A coluna final, F, é o resultado da operação XOR (⊕) entre as duas colunas intermediárias que acabamos de criar. Lembre-se, XOR é 1 apenas quando as entradas são diferentes.

Da Tabela-Verdade para a Expressão Lógica

A tabela-verdade também pode ser usada no sentido inverso: se você tem uma descrição funcional de um circuito (ou seja, uma tabela-verdade completa), você pode derivar a expressão booleana correspondente. O método mais comum é a Soma de Produtos (Sum of Products - SOP).

  1. Foque nas Saídas 1: Encontre todas as linhas na tabela-verdade onde a saída final (F) é igual a 1.
  2. Crie os Termos de Produto (Minterms): Para cada uma dessas linhas, escreva um termo AND (um produto) que só será verdadeiro para aquela combinação específica de entradas. Para fazer isso, se uma variável de entrada é 1, use-a como está. Se for 0, use sua forma negada ().
  3. Some os Termos: A expressão final é a soma (operação OR) de todos os termos de produto que você criou.

Exemplo: Vamos usar a tabela do circuito XOR.

As saídas 1 ocorrem na segunda e terceira linhas. O termo para a segunda linha é ¬AB (pois A=0 e B=1). O termo para a terceira linha é A¬B (pois A=1 e B=0). Somando-os, obtemos a expressão canônica para o XOR: F = ¬AB + A¬B.

Esta expressão pode então ser usada para construir o circuito ou ser simplificada usando teoremas booleanos ou Mapas de Karnaugh.
